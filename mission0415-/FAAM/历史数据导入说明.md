# FAAM 历史数据导入指南

## 功能说明

历史数据导入工具可以帮你将已有的Excel商品数据文件批量导入到FAAM数据库中,并在Web平台上显示。

## 支持的Excel格式

- `.xlsx` (Excel 2007+)
- `.xls` (Excel 97-2003)

## 支持的列名

工具会自动识别以下列名(不区分大小写):

### 必需列
- **TCIN** - 商品唯一标识

## 使用方法

### 方法1: 使用批处理脚本(推荐)

1. **双击运行** `import_history.bat`

2. 选择导入方式:
   - 选项1: 自动扫描FAAM目录及子目录下的所有Excel文件
   - 选项2: 手动指定Excel文件所在目录

3. 等待导入完成,查看统计报告

### 方法2: 命令行运行

```bash
# 导入当前目录
python import_history.py

# 导入指定目录
python import_history.py "D:\Your\Excel\Folder"
```

### 方法3: Python直接调用

```python
from import_history import HistoryDataImporter

importer = HistoryDataImporter()
importer.run("D:/Your/Excel/Folder")
```

## 导入规则

### 1. 新增 vs 更新
- **新增**: TCIN在数据库中不存在 → 插入新记录
- **更新**: TCIN已存在 → 更新现有记录
- **跳过**: TCIN为空或无效 → 跳过该条记录

### 2. 新品标记
- 历史数据导入时,**默认不标记为新品** (is_new = 'No')
- 只有自动化爬虫每天抓取的数据才会被标记为新品

### 3. 数据处理
- 自动去除空白字符
- 自动处理空值(NULL)
- 自动映射常见列名变体
- 保留原有数据的时间戳

## 常见问题

### Q1: 提示"找不到TCIN列"?
**原因**: Excel文件中没有TCIN列或列名不匹配

**解决**:
1. 检查Excel第一行是否包含"TCIN"列
2. 确保列名没有多余空格
3. 可以尝试将列名改为"TCIN"

### Q2: 导入速度慢?
**原因**: 数据量大或Excel文件多

**解决**:
- 耐心等待,工具会显示进度
- 可以分批导入大量文件
- 关闭其他占用资源的程序

### Q3: 部分数据导入失败?
**原因**: 数据格式问题

**解决**:
1. 查看日志了解具体错误
2. 检查Excel文件格式
3. 确保数据类型正确(数字列不要有文字)

### Q4: 如何确认导入成功?
**检查**:
1. 查看最后的统计报告
2. 访问Web平台查看商品数量
3. 搜索已知TCIN确认数据存在

### Q5: 导入后可以删除Excel文件吗?
**回答**: 
- 可以,数据已经保存到数据库
- 但建议保留原始文件作为备份
- 或者移动到archive目录归档

### Q6: 可以重复导入同一个文件吗?
**回答**: 
- 可以,工具会自动识别并更新
- 不会造成数据重复
- 但会浪费时间,建议避免

## 最佳实践

### 1. 导入前准备
- 整理Excel文件到一个目录
- 检查列名是否规范
- 清理明显错误的数据

### 2. 分批导入
如果文件很多(>100个):
- 按日期分批导入
- 每次导入后检查日志
- 确认无误后再导入下一批

### 3. 数据备份
导入前备份数据库:
```bash
copy faam_products.db faam_products_backup.db
```

### 4. 定期清理
- 每月清理一次日志文件
- 保留最近30天的日志
- 归档旧的Excel文件
